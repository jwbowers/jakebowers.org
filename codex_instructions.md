# Codex Instructions for jakebowers.org

This file explains the structure and workflow of the `jakebowers.org` website
so that an AI coding assistant (Codex, Copilot, etc.) can safely maintain it.

## Project overview

- This is a **static website**, generated locally using Python.
- HTML files are generated by `generate_site.py`.
- The site is deployed to **GitHub Pages** via GitHub Actions.
- The `main` branch contains:
  - source data
  - templates
  - CSS
  - the generator script
- The `gh-pages` branch contains **only generated HTML**.

## Key files and directories

### Data (edit these most often)

- `data/vita.bib`
  - BibTeX file for publications.
  - Working papers are marked as `@unpublished`.
- `data/projects.yaml`
  - Current projects, backburner projects, software.
- `data/teaching.yaml`
  - Courses and syllabus links.
- `data/bio.md`
  - Front-page biography text.
- `data/config.yaml`
  - Site and author name.
- `data/publication_pdfs.yaml`
  - Map BibTeX keys to local PDF filenames in `static/papers`.

### Templates

- `templates/layout.html`
  - Base layout (header, nav, footer).
- `templates/index.html`
- `templates/publications.html`
- `templates/projects.html`
- `templates/teaching.html`

Templates are rendered using **Jinja2**.

### Styling

- `static/css/style.css`
  - Single global stylesheet.
  - Design is intentionally minimal and inspired by:
    https://lukeo.cs.illinois.edu/

### Generator

- `generate_site.py`
  - Reads data + templates
  - Writes:
    - `index.html`
    - `publications.html`
    - `projects.html`
    - `teaching.html`
  - Includes a `safe_year()` helper to handle BibTeX years like `{2006}`

⚠️ Do not remove `safe_year()` — it prevents crashes.

## Local workflow

After **any change** to data, templates, or CSS:

```bash
python generate_site.py
```
